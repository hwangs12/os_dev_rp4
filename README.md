# Why another repo for os development when there are plenty out there
It turns out understanding is not an easy task. You have reference manual, which is just a thick book and you are not sure why the bit manipulations are written in certain way. This repo is to help you understand why the code is the way it is and I'd like to engage and expand more by writing here and learn as well during this process. 

Also, the os development guide that I'm following is mainly written for raspberry pi 3. Certain base addresses are different in raspberry pi 4, which is the gear that I have. I feel grateful that I'm using different version because otherwise I would have blindly followed the guide without trying to understand the code underneath. Why different base addresses matter? And, where should we go to find this information? We will cover those here. The main guide I'm following is https://github.com/s-matyukevich/raspberry-pi-os. 

## Setup
I have Macbook laptop, screen cli installed, USB to TTL serial cable. Again, I am following this repo https://github.com/s-matyukevich/raspberry-pi-os for setup and basic code. Some tips: if you have macbook like me, you can ignore the USB to TTL connection to guide bit where it says install driver like *prolific* or *silabs*. I don't think you need to, screen works fine as it is.
